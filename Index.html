OpenGl Implementation

An introduction to OpenGL:-

Open Graphics Library (abbreviated OpenGL) is an Application Program Interface(API) used for rendering 2D and 3D graphics. The code for drawing a particular image is usually written in C or C++.

Standard Header files used and their functions

Iostream.h - provides basic input and output services. Contains the definition for functions like cout, cin.
Windows.h - Windows specific header file which contains the declarations of all functions in Windows API. Contains a variety of child header files like string.h, ctype.h, except.h, which are commonly used in C++ programs. Useful since OpenGL comes under the Win32 apps
GL/gl.h, GL/glu.h, GL/glaux.h - Contains the definition of the openGL inbuilt functions like glutinit, glcolor3f(), glvertex21(), etc.

Description of the image we tried to animate on OpenGL

The image which we tried to realise on OpenGL is that of a compound wall built along the street, which the main characters run across. We have used the line drawing algorithm to draw the edges of this wall, and the portion of the road around it. Color filling has been done accordingly

A brief description of some of the important OpenGL in-built functions used in our code

glOrtho() - sets the display area dimensions, where our image will be rasterized

glPointSize() - specifies the diameter of the rasterized point. A bigger diameter would mean a thicker point, and hence a thicker line/circle/polygon

glColor3f(red,blue,green) :- used for filling colour in any given diagram. As we vary the values of red, blue and green, we go across various colour combinations

glBegin() - Declared before we start drawing a polygon

glEnd() - Declared after the polygon has been drawn, in order to terminate the process and start a new one

glFlush() - Returns the drawn figure to the output buffer, so that it can be projected onto the screen
